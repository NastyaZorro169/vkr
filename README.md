# Код, файлы, результаты для ВКР

Проект содержит код и результаты экспериментов для выпускной квалификационной работы по классификации и сегментации зон затопления на основе спутниковых снимков Sentinel-2.

Основные задачи проекта:

* подготовка спутниковых данных;
* формирование обучающей и валидационной выборок;
* обучение моделей бинарной классификации затопления;
* тестирование методов сегментации водной поверхности;
* выделение водной области и береговой линии.

## Описание файлов

### `classification_preprocessing.py`

Скрипт предварительной обработки данных для задачи бинарной классификации затопления.

Скрипт выполняет:

* чтение исходных спутниковых данных Sentinel-2;
* отбор сцен с полным покрытием;
* чтение спектральных каналов;
* расчет индекса NDWI;
* формирование трехканального входного массива;
* сохранение подготовленных данных;
* формирование файлов `train.csv` и `val.csv`.

Результаты сохраняются в:

```text
data/classification_preprocessed/
```

### `classification_training.ipynb`

Ноутбук для обучения и сравнения моделей классификации.

В нем выполняется:

* загрузка подготовленных данных;
* обучение CNN-моделей;
* подбор гиперпараметров;
* расчет метрик Accuracy, Precision, Recall, F1-score и ROC-AUC;
* сохранение графиков, метрик и результатов экспериментов.

Результаты сохраняются в:

```text
data/classification_runs/
```

### `test_methods.ipynb`

Ноутбук для тестирования методов сегментации водной поверхности.

В нем выполняется проверка различных подходов к выделению водной области, включая методы на основе спектральных индексов, кластеризации и морфологической обработки.

### `segmentation.ipynb`

Ноутбук для итоговой сегментации водной поверхности и выделения береговой линии.

В нем выполняется:

* загрузка спутниковых снимков;
* расчет спектральных индексов;
* сегментация водной поверхности;
* постобработка бинарной маски;
* выделение границ водного объекта;
* визуализация результатов.

## Папка `data`

### `data/classification_preprocessed/`

Папка содержит подготовленные данные для классификации.

В репозитории хранятся:

```text
train.csv
val.csv
S2list_extended.json
```

Файлы `.npy` с изображениями не хранятся в GitHub, так как они занимают большой объем. Их нужно сгенерировать локально после скачивания исходного датасета и запуска preprocessing-скрипта.

Ожидаемая локальная структура после preprocessing:

```text
data/classification_preprocessed/
├── images/
├── train.csv
├── val.csv
└── S2list_extended.json
```

### `data/classification_runs/`

Папка содержит результаты экспериментов классификации:

* конфигурации запусков;
* значения метрик;
* графики обучения;
* confusion matrix;
* CSV-файлы с предсказаниями;
* итоговые сводные таблицы.

## Скачивание датасета

Исходные данные не хранятся в репозитории из-за большого размера.

Ссылки для скачивания датасета:

```text
Датасет SEN12FLOOD - размеченный набор данных,который был найден на просторах интернета

(https://disk.yandex.ru/d/XtC36k5O_71mBQ)

Датасет SEN-MZYMTA-FLOOD - набор данных, который был скачан мною реки Мзымта и её окрестностей

[ВСТАВИТЬ ССЫЛКУ НА ДАТАСЕТ 2]


```

После скачивания SEN12FLOOD нужно распаковать в папку (достав именно что внутри папки):

```text

```

```text

```

Итоговая локальная структура данных должна быть примерно такой:

```text
data/
├── raw/
│   └── [SEN12FLOOD папки со снимками и json с информацией о папках]

│   └── [SEN-MZYMTA-FLOOD папки со снимками и json с информацией о папках]

```

## Создание виртуального окружения

Рекомендуется запускать проект через виртуальное окружение `venv`.

### Windows PowerShell

```powershell
py -3.11 -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
```

Если PowerShell запрещает запуск скриптов, выполните:

```powershell
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
.\.venv\Scripts\Activate.ps1
```

## Установка зависимостей

После активации виртуального окружения установите зависимости:

```powershell
python -m pip install -r requirements.txt
```

Проверить, что окружение активно:

```powershell
python --version
python -m pip --version
```

## Порядок запуска проекта

Проект запускается в следующем порядке:

1. Предобработка данных.
2. Обучение классификационных моделей.
3. Тестирование методов сегментации.
4. Итоговая сегментация.

## 1. Предобработка данных

Перед запуском убедитесь, что исходные снимки лежат в:

```text
data/raw/
data/mzymta/
```

Запуск:

```powershell
python classification_preprocessing.py
```

После выполнения должны появиться подготовленные данные:

```text
data/classification_preprocessed/train.csv
data/classification_preprocessed/val.csv
data/classification_preprocessed/images/
```

## 2. Обучение классификационных моделей

Откройте ноутбук:

```text
classification_training.ipynb
```

Запускать ячейки нужно последовательно сверху вниз.

Перед запуском убедитесь, что существуют:

```text
data/classification_preprocessed/train.csv
data/classification_preprocessed/val.csv
data/classification_preprocessed/images/
```

Результаты обучения сохраняются в:

```text
data/classification_runs/
```

## 3. Тестирование методов сегментации

Откройте ноутбук:

```text
test_methods.ipynb
```

Запускать ячейки нужно последовательно сверху вниз.

Этот ноутбук используется для проверки разных методов выделения водной поверхности и сравнения промежуточных результатов.

## 4. Итоговая сегментация

Откройте ноутбук:

```text
segmentation.ipynb
```

Запускать ячейки нужно последовательно сверху вниз.

Ноутбук выполняет итоговую сегментацию водной поверхности, постобработку маски и визуализацию результата.
